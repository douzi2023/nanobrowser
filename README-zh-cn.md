## 🔥 为什么选择 Nanobrowser？

正在寻找功能强大的 AI 浏览器代理，却不想每月为 OpenAI Operator 支付 200 美元吗？**Nanobrowser** 作为一款 Chrome 扩充功能，提供进阶的网页自动化能力，同时让您保有完全的掌控权：

- **100% 免费** - 无订阅费或隐藏成本。安装后使用自己的 API 密钥，只需支付给供应商的 API 使用费。
- **注重隐私** - 所有处理都在本机浏览器内完成。您的认证资讯会储存在本机，绝不会与任何云端服务分享。
- **弹性的 LLM 选项** - 可连接您偏好的 LLM 供应商，并可为不同代理选择不同模型。
- **完全开放源代码** - 浏览器自动化过程完全透明，没有任何暗箱操作或隐藏的处理程序。

> **请注意：** 我们目前支援 OpenAI、Anthropic、Gemini、Ollama、Groq、Cerebras、Llama 以及自订的 OpenAI 相容供应商，未来将会支援更多。


## 📊 主要功能

- **多代理系统**：由专业的 AI 代理协同合作，完成复杂的网页工作流程
- **互动式侧边面板**：直觉的聊天界面，提供即时的状态更新
- **任务自动化**：跨网站无缝自动化重复性任务
- **后续提问**：针对已完成的任务提出与上下文相关的追问
- **对话历史纪录**：轻松存取并管理与 AI 代理的互动历史
- **支援多种 LLM**：可连接您偏好的 LLM 供应商，并为不同代理指派不同模型

## 🔧 手动安装最新版本

若要取得包含所有最新功能的版本：

1. **下载**
    * 从官方 GitHub 的 [版本页面](https://github.com/nanobrowser/nanobrowser/releases) 下载最新的 `nanobrowser.zip` 档案。

2. **安装**：
    * 解压缩 `nanobrowser.zip`。
    * 在 Chrome 中开启 `chrome://extensions/`
    * 启用 `开发人员模式` (右上角)
    * 按一下 `载入未封装的扩充功能` (左上角)
    * 选择已解压缩的 `nanobrowser` 资料夹。

3. **设定代理模型**
    * 按一下工具列中的 Nanobrowser 图示以开启侧边面板
    * 按一下右上角的 `设定` 图示。
    * 新增 LLM API 密钥。
    * 为不同代理 (Navigator、Planner) 选择要使用的模型。

4. **升级**：
    * 从版本页面下载最新的 `nanobrowser.zip` 档案。
    * 解压缩并用新档案覆写您现有的 Nanobrowser 档案。
    * 前往 Chrome 的 `chrome://extensions/` 页面，然后在 Nanobrowser 卡片上按一下重新整理图示。

## 🛠️ 从源代码建置

如果您偏好自行建置 Nanobrowser，请依照以下步骤操作：

1. **先决条件**：
   * [Node.js](https://nodejs.org/) (v22.12.0 或更高版本)
   * [pnpm](https://pnpm.io/installation) (v9.15.1 或更高版本)

2. **复制储存库**：
   ```bash
   git clone https://github.com/nanobrowser/nanobrowser.git
   cd nanobrowser
   ```

3. **安装相依套件**：
   ```bash
   pnpm install
   ```

4. **建置扩充功能**：
   ```bash
   pnpm build
   ```

5. **载入扩充功能**：
   * 建置完成的扩充功能将位于 `dist` 目录中
   * 依照「手动安装」一节中的步骤，将扩充功能载入浏览器

6. **开发模式** (选用)：
   ```bash
   pnpm dev
   ```

## 🤖 选择您的模型

Nanobrowser 允许您为每个代理设定不同的 LLM 模型，以平衡效能与成本。以下是建议的设定：

### 追求高效能
- **Planner**：Claude Sonnet 4
  - 更佳的推理与规划能力
- **Navigator**：Claude Haiku 3.5
  - 有效率地处理网页导览任务
  - 在效能与成本之间取得良好平衡

### 讲求成本效益
- **Planner**：Claude Haiku 或 GPT-4o
  - 以较低成本获得合理的效能
  - 处理复杂任务可能需要更多次的迭代
- **Navigator**：Gemini 2.5 Flash 或 GPT-4o-mini
  - 轻量级且具成本效益
  - 适合基本的导览任务

### 本机模型
- **设定选项**：
  - 使用 Ollama 或其他自订的 OpenAI 相容供应商，在本机执行模型
  - 零 API 成本并确保完全隐私，所有资料都保留在本机电脑

- **推荐模型**：
  - **Qwen3-30B-A3B-Instruct-2507**
  - **Falcon3 10B**
  - **Qwen 2.5 Coder 14B**
  - **Mistral Small 24B**
  - [社群最新测试结果](https://gist.github.com/maximus2600/75d60bf3df62986e2254d5166e2524cb)
  - 欢迎社群成员在我们的 [Discord](https://discord.gg/NN3ABHggMK) 分享其他本机模型的使用经验

- **提示词工程**：
  - 本机模型通常需要更具体、清楚的提示词
  - 避免使用高层次、模糊的指令
  - 将复杂的任务拆解成清楚、详细的步骤
  - 提供明确的上下文与限制条件

> **请注意**：讲求成本效益的设定可能会产生较不稳定的输出，且处理复杂任务时可能需要更多次的迭代。

> **提示**：欢迎尽情尝试自己的模型设定！找到绝佳组合了吗？到我们的 [Discord](https://discord.gg/NN3ABHggMK) 与社群分享，帮助大家最佳化设定。
